# ADR-002: SATAN arrival via metabolic stochastic gating

> yet accepted. It depends on ADR-001 and must clear the open questions

> acceptance.

## Context

ADR-001 decouples perception from cognition: the LLM **consumer** now
arrives on its own trigger, separate from the deterministic perception view.
That makes *consumer arrival* a free variable for the first time — previously it
was welded to the fixed `satan-tick` timer.

`governance.md`'s core thesis lists **invocation schedule** in SATAN's durable
DNA, and `architecture.md` states arrival is *"explicit, scheduled, inspectable,
boring. SATAN does not decide for itself when to wake up except through
mechanisms the user has explicitly installed."* Today that means fixed systemd
timers (`satan-tick OnUnitActiveSec=30min`, `morning`, `motd`).

The attribute layer (`docs/satan/attributes/design-contract.md`) already exists
as **metabolic control variables that bias behaviour** — but only *within* a
run (read from the prompt capsule). It is deterministic, append-only,
event-sourced, capped, and decays daily toward homeostasis. The proposal: let
that same metabolism also decide *arrival*.

## Decision

**PROPOSED — gate consumer arrival on attribute state, stochastically.**

- Arrival is **biased-random**, not fixed-schedule: high Curiosity → likely
  info-gathering run; high Metamorphosis → likely introspection/self-edit;
  Brooding → ruminate. The gate reads attributes, rolls a biased die, maybe
  fires; a completed run of a kind **satiates** its matching attribute
  (integrate-and-fire / leaky bucket → bursty, natural rhythm).
- **Reuses existing machinery**, does not reinvent: satiation already exists
  (source-event tables decrement the matching attribute on the matching action);
  the homeostatic floor already exists (T-attr-2 daily decay); continuous
  integration already exists (sensors charge attributes every tick). The gate is
  read + roll + (on-fire) satiate.
- **Two timescales, shared vocabulary.** Slow *global* attributes drive
  *discretionary* arrival (metabolism). Intervention urgency is a fast,
  cue-bound spike → a **fast episode-scoped transient** attribute drives the
  intervention reflex. The contract already reserves `scope = episode | motive`
  but v1 only writes `global`; this is what motivates implementing it.
- **Placement:** daemon-owned (`satan-attrd`, consistent with decay) — reads
  attrs, rolls, signals the broker to spawn via the existing pg_notify/inbox
  path. May retire the systemd heavy-run timer; the waybar widget becomes manual
  arrival injection.

Capture: IMPR-012. Builds on ADR-001 / DE-010.

## Consequences

### Positive

- Arrival texture matches the organism metaphor (`patterns_attributes`:
  "global attributes are the animal's metabolism"). Self-paced, non-mechanical.
- Almost entirely reuses the event-log / caps / replay / decay substrate.
- Subsumes the fixed-timer zoo into one principled mechanism.

### Negative — these are why this is `proposed`, not `accepted`

- **Contradicts the current DNA.** Shifts arrival deterministic-schedule →
  self-paced-stochastic. `governance.md` + `architecture.md` say arrival is
  "explicit, scheduled, boring." Accepting this ADR **requires amending** that
  doctrine, not quietly overriding it.
- **Self-manipulation surface.** The model already nudges its own attributes via
  tool calls (design-contract §6H.7). Once attributes gate *arrival*, indirect
  self-summoning becomes possible. **A written self-manipulation analysis is a
  precondition of acceptance.**
- **Replay-determinism risk.** The attribute layer's foundation is byte-exact
  replay (`Math.random`/`Date.now` are banned there). An unlogged dice roll
  breaks it.

### Neutral

- A new dispatcher **source** (`source=arrival`/`gate`) with its own reason enum
  + delta table + caps; slots into the contract's reserved-source pattern.

## Guards (mandatory if accepted)

1. **Daily token ceiling is the hard cap** (`dl-satan-budget`) — a saturated
   metabolism cannot summon unbounded runs.
2. **Fire-probability modulated by remaining budget** — hungry + broke →
   suppressed.
3. **Rolls are seeded + logged** as an `arrival.rolled` event (threshold, roll,
   fired?) so arrival is as auditable as every attribute delta.
4. **Probability, not trigger** — attributes bias odds; they never force a run.

- `arrival.rolled` events present + replay-deterministic (seeded).
- Arrival count over a window respects the budget ceiling under saturation.
- Self-manipulation: model-driven attribute nudges cannot raise arrival rate

## Self-manipulation analysis (acceptance deliverable 1)

### Threat model

The model is untrusted. Attributes will gate arrival probability. Therefore
any channel by which the model influences attribute state becomes a channel
for **indirect self-summoning**: raising its own invocation rate, spending
keeper tokens and attention without keeper intent.

**Model-writable channels** (ordered by strength):

1. Tool calls with attribute side effects — hippocampus writes (§6H,
   0.025/call, ≤30 calls/run ceiling).
2. Intervention choices — the model chooses whether/what to intervene,
   knowing outcome events charge attributes.
3. Memory-shaping — content the model writes today shapes future resonance
   matches; if/when the reserved `resonance` source is wired, this becomes an
   indirect delta channel.
4. Motive file writes — bounded by motive validation; not attribute-wired
   today.

**Not model-writable:** sensor deltas (external world — the model cannot type
for the keeper), daemon decay, keeper manual marks, quiet-hours gate, enable
switches, budget ceiling.

### Why the loop is contained — five layers

**L1 — Capability neutrality.** An extra arrival buys a run inside the same
broker membrane: same mode/tool allowlists, same audit, same validation. No
permission the model lacks at 2 runs/day appears at 20. Self-summoning
escalates *frequency*, never *capability*. This is the frame for everything
below: the threat is resource nuisance, not authority escalation.

**L2 — Hard resource bound (Guard 1 + 2).** The daily token ceiling is
attribute-independent and broker-owned; a saturated metabolism reallocates
the day's budget, it cannot expand it. Budget-modulated fire probability
(Guard 2) adds second-order negative feedback: the faster the model spends,
the harder arrival suppresses. Worst case, absolute: the model burns one
day's budget doing more runs of legitimately-permitted work.

**L3 — Subcriticality (new invariant, proposed Guard 5).** The gate's design
already contains its own damping: **satiation** — a completed run of kind X
decrements X. The self-summoning loop (run → inflate → likelier arrival →
run) is a branching process; it dies out if each cycle loses more charge
than the model can re-inject:

```
satiation(X) + decay(X) > max per-run model-writable inflation(X)
```

Today's magnitudes satisfy this with margin: max hippocampus inflation is
0.025/call against a 30-call budget that useful work also consumes
(realistic writes ≤ a handful), while satiation is per-fire and decay is
daily. **Adopt the inequality as a standing invariant checked at every
contract amendment** — any new model-writable source or magnitude change
must keep every arrival-relevant attribute subcritical (R < 1). Verifiable
by table inspection plus the adversarial test in §Verification.

**L4 — Saturation.** Values clamp to [0,1] and caps (§7, snapshot
semantics) bound deltas; probability per kind has a ceiling < 1 (Guard 4:
bias, never force). Inflation asymptotes; it cannot spike.

**L5 — Detection.** Every roll is seeded and logged (`arrival.rolled`:
threshold, roll, fired) beside the append-only `attribute.delta_applied`
stream. One query reconstructs *why* any arrival happened and which sources
charged it. Sustained gaming shows as an arrival-rate histogram anomaly with
a fully attributed cause chain. (Natural rider on SL-011 phase-3 alerting:
alert on arrivals/day > N.)

### Residual risks and rulings

- **Cross-kind collusion** — inflate several attributes so *some* gate
  fires: per-kind subcriticality doesn't bound the sum. **Proposed Guard 6:
  global daily arrival-count cap** (independent of, and cheaper than, the
  token ceiling). Sum-over-kinds bounded by construction.
- **Episode-scoped transients** (the fast intervention reflex, D2-hybrid):
  spikes are meant to be cue-bound. **Proposed rule: arrival-relevant
  episode-scoped attributes accept only deterministic sources** (sensor,
  broker, outcome projection) — never model tool calls. Keeps the reflex
  channel out of the model's reach entirely.
- **Slow drift below alert threshold** — accepted: payoff is capped by L2,
  cost grows via L3, and the audit trail makes retrospective review cheap.
- **Precedent check** (§6H.7 analogue): the model already influences
  attributes for in-run flavor; arrival raises the stakes from "what mood is
  the capsule in" to "when do I run", which is exactly why this analysis
  exists. The containment is structural (L1–L4), not behavioural trust.

### Conclusion

With existing Guards 1–4 plus proposed Guards 5 (subcriticality invariant)
and 6 (daily arrival cap) and the transient-source restriction,
self-summoning is capability-neutral, resource-bounded, subcritical, and
detectable. The doctrine amendment below can honestly retain "inspectable
and bounded" while dropping "fixed-schedule".

## Doctrine amendment text (acceptance deliverable 2)

Apply on acceptance, not before.

### `docs/satan/architecture.md` — Invocation layer, replacement text

> ### Invocation
> Defines when and why SATAN runs. Two arrival modes, both user-installed:
>
> - **Scheduled entries** — deterministic calendar arrivals (morning,
>   MOTD, weekly review) and manual invocation (waybar, `M-x`). Explicit,
>   inspectable, boring.
> - **Metabolic gate** (ADR-002) — discretionary arrival gated on attribute
>   state, stochastically: the daemon reads attributes, rolls a seeded and
>   logged die, and may signal the broker to spawn. Bias, never trigger.
>   Bounded by the daily token ceiling, budget-modulated probability, a
>   daily arrival-count cap, and the subcriticality invariant on
>   model-writable attribute sources. Every roll is auditable
>   (`arrival.rolled`); disabling the gate falls back to the scheduled
>   timer.
>
> SATAN does not decide for itself when to wake up except through
> mechanisms the user has explicitly installed — the metabolic gate is such
> a mechanism, with its ceilings and kill switch owned outside the model's
> reach. Arrival is no longer fixed-schedule; it remains explicit in
> policy, inspectable in audit, and bounded in resources.

### `docs/satan/governance.md` — core thesis, DNA list amendment

Replace the DNA line `+ invocation schedule` with:

> + invocation policy   (scheduled entries + metabolic gate parameters,
>                        ceilings, and kill switch)

with a one-line footnote at first use:

> *Invocation policy* supersedes *invocation schedule* (ADR-002): SATAN's
> durable identity includes **how arrival is governed** — the gate's
> guards, ceilings, and audit — not a particular timer table.
