# RFC-016: SATAN evolution: earn attention, close the loop

> **Imported 2026-07-22** from the `.emacs.d` doctrine corpus, frozen for SATAN.
> Ids cited below resolve in that corpus unless they also exist here; `DE-`/`DR-`
> have no doctrine equivalent, and `IMPR-NNN` / `ISSUE-NNN` are pre-doctrine
> prefixes for `IMP-NNN` / `ISS-NNN`.

## Context

Companion to RFC-001 (architectural rehoming). That RFC asks where SATAN
lives; this one asks what it should *become*. Grounded in a four-way audit
(2026-07-06): design docs vs completion state, doctrine entity state, both
satellite repos, and runtime liveness.

### State of the organism

| Layer | State |
|---|---|
| Perception (percept, resonance, motive, sensors) | Built, shipped, **alive** — continuous segment coverage 2026-05-28 → present, zero gaps. The most robust layer. |
| Metabolism (attributes, satan-attrd) | Built and running (hourly decay), but per `wiring-status.md` half its levers are unreachable: Friction/Cruelty structurally locked at 0, Suspicion near-dormant, sources `percept`/`resonance`/`tool_error`/`manual` reserved-unwired. *(Doc may lag reality — verify before acting.)* Attribute events since ~06-23 are a flat 4–6/day decay floor. |
| Cognition (tick → LLM runs) | **Dormant since ~2026-06-23.** Every fired run dies at HTTP 402 (OpenRouter credits). Waybar timer is the effective trigger; systemd `satan-tick` fires once per boot only. |
| Behaviour (interventions, daily block, MOTD, notifications) | Last intervention **2026-05-31**. Output surface dead ~5 weeks. |
| Epistemics (hypothesis, probe, falsification) | **0% built.** The roadmap's own words: currently "the context-aware nudge machine §3 warns against… but nothing to test." |
| Patch agent | Mechanism landed; **never one clean end-to-end acceptance run** (17/17 jobs failed; known `jailed-pi` PATH blocker; runner-extraction pivot undecided). |

Governance context: only perception carries an accepted decision (ADR-001,
partially realized — IMP-013 and SL-010/P02 outstanding). ADR-002 (metabolic
arrival gating) proposed and blocked ~1 month on two work items. Attributes,
memory, observer, intervention, patch are designed in markdown outside
doctrine; **zero specs exist**. (Absence of audits/closures is spec-driver →
doctrine migration residue, not process failure — there has been no
doctrine-native SATAN development yet.)

### The corrected diagnosis

The dormancy is not an infrastructure failure. 1Password unlocks reliably;
the waybar trigger works; credits are a top-up away. **SATAN went dormant
because its keeper stopped caring to interact — and the organism cannot
perceive that.** The outcome observer is deliberately positive-only (no
`ignored`, no `inconclusive`; Q7 punted in `perceptual-design.md`). Five
weeks of total disengagement — the most informative behavioural signal the
system has ever produced — was unrepresentable in its own substrate. The
metabolism decayed quietly toward baseline, and with ADR-002 unaccepted,
even that decay had no behavioural consequence.

An organism whose survival depends on being interesting, with no sense organ
for being ignored, and no consequence when it is.

### Why the build order got here

Mechanism outran utility. Enormous machinery (memory substrate, percept
capsule, attribute layer, intervention records, patch scaffolding) serves
utility surfaces (daily block, MOTD, nudges, review flow) that stayed thin.
The governance doc's own maxims — "a good daily block beats a half-trusted
general agent"; "avoid adding autonomy before improving review, audit, and
context" — name the priority the build order inverted.

## Discussion

### D1 — The organizing principle: earn attention, and notice when you don't

Every phase below is judged against one question: *does this make SATAN more
worth interacting with, or make it adapt when it isn't?* Structural work
(RFC-001 rehoming, POL-001 extractions) rides along behind utility work,
never leads it.

### D2 — Phase 0: resurrect (days)

- Top up OpenRouter credits.
- Clear/repair the attrd poison message (outcome_inbox row id=989, schema-
  drifted `sensor: content_backlog` payload rejected on every startup).
- Optional: reconcile the systemd `satan-tick` timer with the waybar trigger
  (one intentional cadence, not two half-owners).

Nothing else matters while cognition cannot complete a run.

### D3 — Phase 1: close the smallest loop (weeks)

Keystone: **disengagement becomes perceptible and consequential.**

- `ignored` / absent-outcome as a first-class observer verdict (revisits the
  positive-only predicate and Q7 attribution).
- Disengagement as attribute stimulus — natural first use of the unwired
  `manual` / `percept` sources (verify wiring-status.md first).
- **Unblock ADR-002**: write the self-manipulation containment analysis and
  the governance.md/architecture.md amendments. Accepting it gives attributes
  their first behavioural consequence — being ignored finally *costs
  something* (quieter, differently-voiced, or hungrier arrival).
- Make the daily block excellent again — it is the retention surface the
  maxims already prioritize.
- SL-011 (tick perf observe+bound) executes in parallel; independent.

### D4 — Phase 2: epistemics MVP (the differentiator)

The engagement engine. One hypothesis kind, one probe surface (a question in
the daily block suffices), falsifiable, outcomes feeding the already-landed
pattern/scar records (DE-009). Deliberately tiny: the goal is the first
closed suspicion → probe → verdict loop, not the epistemics roadmap's full
grammar. A system that forms a testable suspicion about its keeper and is
visibly wrong sometimes gives the keeper a reason to look; nudges never
will.

### D5 — Phase 3: patch agent — decide the pivot before fixing it

Do not fix the `jailed-pi` PATH blocker inside the elisp runner if IMP-006
(extraction to satan-patcher) says the runner is leaving — POL-001 trigger 2
(recurring runtime-fit failure) has arguably already fired. Decide
extraction first; make `patch/brief.md` §4.6 acceptance the revival gate.

### D6 — Product spec as the anchor

First doctrine-native SATAN work: a product spec pinning down what SATAN
does *for the keeper* — the utility surfaces, the engagement loop, success
measures ("did the keeper interact this week" is a legitimate metric).
Every future slice then justifies itself against utility instead of
mechanism. Fixes the altitude problem that let mechanism outrun purpose.

### D7 — Structural work rides along

- RFC-001 rehoming proceeds independently; Phase 2 memory growth is the
  natural IMP-007 (satan-memoryd) trigger per POL-001.
- Doc reconciliation as a chore: stale HANDOVERs in both satellites
  (attrd's says "scaffold only"; it is feature-complete through decay),
  capsule-render contradiction (`epistemics-roadmap.md` vs
  `refactor/plan.md`), wiring-status verification.
- SL-007 (interactive pi.dev harness) — stale 5 weeks; revive or park
  explicitly.

### Alternatives considered

- **Ops-first framing** (fix timers/secrets/monitoring, add a dead-man
  alarm): rejected as the centre — the infra is less brittle than it looks,
  and a reliably-ticking system nobody reads is the same failure at higher
  cost. A cognition-liveness alert may still ride Phase 1 cheaply.
- **Architecture-first** (finish rehoming, then utility): inverts the lesson;
  rehoming does not make SATAN worth reading.
- **Full epistemics build-out** per roadmap: too big; the MVP loop first,
  grammar after evidence.

### Crystallization path

- ISSUE: cognition dormant — 402 + poison row (Phase 0 umbrella).
- IMP: `ignored` outcome + disengagement sensing (Phase 1 keystone).
- IMP: attribute source wiring (`manual`, `tool_error`, then
  `percept`/`resonance`) — after wiring-status verification.
- ADR-002: unblock work items, then accept/reject.
- SPEC (product): SATAN utility + engagement loop (D6).
- IMP: epistemics MVP (D4) — or a slice directly once the product spec
  frames it.
- Decision on IMP-006 pivot before any patch-runner fix (D5).
- CHORE: satellite/doc reconciliation (D7).

## Outcome

Open. Framing agreed with the keeper (2026-07-06): dormancy is an
attention-economy failure, not an ops failure; evolution is judged on
earning interaction and sensing its absence. Unresolved:

- verify wiring-status.md against `dl-satan-attribute.el` before scoping the

- ADR-002 acceptance (pending its two blocking work items);
- whether epistemics MVP is scoped as backlog item or direct slice under the
  product spec;
- patch runner: extract-then-fix vs fix-in-place (D5 leans extract).
